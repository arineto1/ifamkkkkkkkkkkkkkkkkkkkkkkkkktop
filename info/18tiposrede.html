01 Classificação quanto à Arquitetura

Cliente-Servidor (Client-Server)

Baseada na assimetria de funções. O Servidor é um host que fornece recursos ou serviços, enquanto o Cliente os requisita.

  • Servidor: Sempre ligado, IP fixo, passivo (espera requisições).
  • Cliente: Ativo (inicia comunicação), IP dinâmico, não fala diretamente com outros clientes.
  • Peguinha: Centralização = Ponto Único de Falha. Se o servidor cai, a rede para.

Ponto a Ponto (P2P - Peer-to-Peer)

Os nós são Equipotentes (Peers). Cada dispositivo pode atuar como cliente e servidor simultaneamente (Servent).

  • Escalabilidade: Auto-escalável. Quanto mais nós, maior a capacidade de processamento/armazenamento.
  • Exemplos: BitTorrent, Skype (antigo), Blockchain.

03 Topologia de Rede

Diferencie Física (layout real dos cabos) de Lógica (fluxo dos dados).

Barramento (Bus)

Um único cabo compartilhado. Se houver falha no cabo central (backbone), toda a rede cai. Usa terminadores.

Anel (Ring)

Unidirecional. Usa o Token Passing. Ponto de falha crítico: se um nó falha, o anel rompe (salvo anéis duplos FDDI).

Estrela (Star)

Mais comum hoje. Nó central (Switch). Se um host falha, a rede continua. Se o concentrador central falha, a rede para.

Malha (Mesh)

Totalmente conectada. Redundância máxima.
Fórmula de links: n(n-1)/2.

04 Meios de Transmissão e Fluxo

Direcionalidade (Modos de Transmissão)

  • Simplex: Unidirecional (ex: TV, Rádio).
  • Half-Duplex: Bidirecional, não simultâneo (ex: Walkie-talkie, Hub).
  • Full-Duplex: Bidirecional e simultâneo (ex: Telefone, Switch).

Meios Guiados (Cabeamento)

Par Trançado: UTP (sem blindagem), STP (com blindagem).
Cat 5e (1Gbps), Cat 6 (10Gbps curta), Cat 6a (10Gbps 100m).

Fibra Óptica: Imune a ruídos eletromagnéticos.

  • Monomodo (SMF): Um só raio de luz, longa distância, caro.
  • Multimodo (MMF): Vários raios, distâncias curtas, mais barato (LED).

Radar CEFETMINAS: O que derruba candidatos

1. Topologia Estrela vs Barramento: Em uma estrela usando HUB, a topologia física é estrela, mas a lógica é barramento (pois o Hub replica para todos). Se usar Switch, a lógica também se torna estrela/ponto-a-ponto.

2. Domínios: Hub compartilha 1 domínio de colisão. Switch cria 1 domínio de colisão por porta. Ambos mantêm apenas 1 domínio de broadcast (a menos que usem VLAN).

3. SAN vs NAS: A banca pode dizer que SAN é um hardware. Falso: SAN é uma rede de armazenamento. NAS é o dispositivo ligado na rede.
