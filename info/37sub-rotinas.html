1. Modularização e Definição

A utilização de sub-rotinas baseia-se no princípio de "Dividir para Conquistar". Consiste em decompor um problema complexo em partes menores, independentes e reutilizáveis.

Características Essenciais:
  • Reutilização: O código é escrito uma vez e invocado múltiplas vezes.
  • Abstração: O chamador sabe o que a sub-rotina faz, mas não necessariamente como (encapsulamento).
  • Facilidade de Manutenção: Alterações são feitas em um único ponto do código.
Tipo Descrição Técnica Exemplo de Uso
Procedimento (Procedure) Realiza uma tarefa sem retornar um valor associado ao seu nome. Em C, usa-se void. limpar_tela(), ordenar(vetor)
Função (Function) Calcula e retorna obrigatoriamente um valor para o ponto de chamada. resultado = calcular_raiz(16)

2. Escopo e Ciclo de Vida

O "Escopo" determina a visibilidade e a validade de uma variável dentro do programa.

  • Variáveis Globais: Declaradas fora de qualquer sub-rotina. Visíveis em todo o programa. Perigo: Dificultam o rastreamento de erros e ferem a modularidade.
  • Variáveis Locais: Declaradas dentro de uma sub-rotina. Existem apenas enquanto a sub-rotina está em execução.
  • Parâmetros Formais: Variáveis declaradas no cabeçalho da sub-rotina que recebem os dados de entrada.
  • Parâmetros Reais (Argumentos): Os valores ou variáveis passados no momento da chamada da sub-rotina.

3. Mecanismos de Passagem (Foco CEFETMINAS)

A banca adora cobrar a diferença prática através de testes de mesa. Memorize as mecânicas:

Por Valor (Cópia)

A sub-rotina recebe uma cópia do valor da variável original. Alterações locais NÃO afetam o original.

void soma(int a) { a = a + 1; } // Chamada: x = 10; soma(x); // x continua sendo 10.

Por Referência (Endereço)

A sub-rotina recebe o endereço de memória da variável. Alterações locais AFETAM o original (efeito colateral).

void soma(int *a) { *a = *a + 1; } // Chamada: x = 10; soma(&x); // x agora é 11.
Peguinha recorrente (Vetores): Em linguagem C, ao passar um vetor para uma sub-rotina, ele é passado automaticamente por referência (passa-se o endereço do primeiro elemento). Qualquer alteração no vetor dentro da função altera o vetor original.

4. Recursividade

Uma sub-rotina é recursiva quando invoca a si mesma. Elementos obrigatórios para a prova:

  • Caso Base (Condição de Parada): Sem ele, ocorre recursão infinita e estouro de pilha (Stack Overflow).
  • Caso Geral (Passo Recursivo): A chamada que aproxima o problema do caso base.
  • Pilha de Execução: Lembre-se que cada chamada recursiva consome memória na pilha (Stack) para armazenar variáveis locais e endereço de retorno.
