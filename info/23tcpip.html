1. Definição e Arquitetura TCP/IP

A arquitetura TCP/IP (Transmission Control Protocol/Internet Protocol) é o conjunto de protocolos que define como os dados devem ser formatados, endereçados, transmitidos, roteados e recebidos. Diferente do modelo OSI, que é um modelo de referência conceitual, o TCP/IP é um modelo prático (de implementação).

Ponto de Prova: O modelo original (DoD - Department of Defense) possui 4 camadas. O modelo acadêmico atualizado possui 5 camadas. A CEFETMINAS costuma cobrar a distinção entre as camadas de Acesso à Rede (4 camadas) versus Enlace/Física (5 camadas).
Camada (5 Cam.) PDU (Unidade) Principais Protocolos
Aplicação Dados HTTP, DNS, SMTP, FTP, SNMP, SSH
Transporte Segmento (TCP) / Datagrama (UDP) TCP, UDP, SCTP
Rede (Internet) Pacote / Datagrama IP IP (v4/v6), ICMP, ARP, RARP, IGMP
Enlace Quadro (Frame) Ethernet, PPP, Wi-Fi (802.11)
Física Bits Sinais elétricos, ópticos, rádio

2. Camada de Transporte: Confiabilidade vs. Velocidade

Esta camada é responsável pela comunicação fim-a-fim. O foco aqui é o multiplexação através de Portas.

  • TCP (Transmission Control Protocol): Orientado à conexão, confiável, garante a entrega e a ordem. Utiliza o Three-way Handshake (SYN → SYN+ACK → ACK). Controle de Fluxo: Usa Janela Deslizante. Controle de Congestionamento: Slow Start e Congestion Avoidance.
  • UDP (User Datagram Protocol): Não orientado à conexão, não confiável ("Melhor Esforço"). Sem confirmação de entrega. Baixa latência. Ideal para VoIP, Streaming e DNS.

3. Endereçamento e Camada de Rede

O IP (Internet Protocol) é o coração desta camada. Ele é responsável pelo endereçamento lógico e roteamento.

IPv4 (32 bits)

Dividido em classes (A, B, C, D, E) ou via CIDR (ex: /24).

  • Privados (RFC 1918): 10.x.x.x | 172.16.x.x - 172.31.x.x | 192.168.x.x
  • Loopback: 127.0.0.1
  • APIPA: 169.254.x.x (Atribuído quando o DHCP falha).

IPv6 (128 bits)

Escrito em hexadecimal. Não possui Broadcast (usa Anycast e Multicast).

Memorize para a FCM: O cabeçalho IPv6 é fixo (40 bytes) para agilizar o processamento nos roteadores. A fragmentação é feita apenas pelo emissor, nunca pelos roteadores intermediários.

4. Protocolos Auxiliares Essenciais

Protocolo Função Crítica para a Prova
ICMP Mensagens de erro e controle (Ping/Traceroute). Opera na Camada de Rede.
ARP Resolve IP → MAC. Mapeia endereço lógico em físico.
DHCP Alocação dinâmica de IPs. Usa portas 67 (Servidor) e 68 (Cliente).
DNS Resolução de nomes. Usa porta 53 (UDP/TCP).
SNMP Gerenciamento de rede. Porta 161 (Agente) e 162 (Trap).

5. Roteamento: RIP, OSPF e BGP

Roteamento é a escolha do melhor caminho para o pacote.

  • Interior (IGP): Dentro de um Sistema Autônomo (AS).
    • RIP: Vetor de distância. Métrica: Saltos (Máx 15).
    • OSPF: Estado de Enlace. Métrica: Custo (Banda). Algoritmo Dijkstra.
  • Exterior (EGP): Entre Sistemas Autônomos.
    • BGP: Vetor de Caminho. Protocolo da Internet mundial.

6. Tendências e "Peguinhas" CEFETMINAS

A Fundação CEFETMINAS, em provas para Professor EBTT, costuma ser técnica e detalhista.

Checklist de Pegadinhas:

  • Protocolo de Gerenciamento: A banca ama confundir SNMP (Gerência) com SMTP (E-mail).
  • Flags TCP: Lembre-se que o RST (Reset) aborta uma conexão, enquanto o FIN termina uma conexão normal.
  • Camadas: Afirmar que o HTTP roda sobre UDP. Falso (Apenas HTTP/3 usa QUIC/UDP).
  • Endereço MAC: 48 bits (6 bytes). Escrito em hexadecimal. Camada 2.
  • NAT: Serve para economizar IPs públicos, traduzindo IPs privados em um ou mais IPs públicos.
