QUESTÃO 01
A construção de algoritmos exige a compreensão clara de suas propriedades fundamentais. Analise a definição a seguir:
"Um algoritmo é uma sequência ___________ e ___________ de passos que visam atingir um objetivo ou resolver um problema específico, transformando dados de entrada em informações de saída, num tempo finito."

Assinale a alternativa que preenche corretamente as lacunas, de acordo com a literatura clássica de lógica de programação adotada pela Fundação CEFETMINAS.
Resolução Detalhada:

Gabarito: B

Esta questão testa a definição canônica de algoritmo. Segundo a literatura técnica:

  • Sequência Lógica: Os passos devem estar ordenados de forma coerente; passos aleatórios não resolvem problemas de forma determinística.
  • Finitude: Esta é a característica mais crítica. Todo algoritmo deve ter um fim. Um processo infinito não é considerado um algoritmo computacional válido para resolução de problemas padrão.

A alternativa correta preenche as lacunas com "lógica" e "finita". As opções que sugerem "infinita", "aleatória" ou "indefinida" contradizem a própria natureza da computação algorítmica.

QUESTÃO 02

Var
   A, B : Inteiro
   Resultado : Real
Inicio
   A <- 9
   B <- 2
   Resultado <- A / B
   Escreva(Resultado)
Fimalgoritmo
Com base no comportamento padrão dos tipos de dados, assinale a alternativa que apresenta o valor que será exibido ao final da execução.
Resolução Detalhada:

Gabarito: B

A questão avalia o conhecimento sobre tipos de dados e o resultado de operações aritméticas.

  1. Variáveis: A e B são Inteiros; Resultado é Real.
  2. Operação: 9 / 2.
  3. Matemática: A divisão aritmética de 9 por 2 é exatamente 4.5.
  4. Armazenamento: Como a variável de destino (Resultado) foi declarada como Real, ela tem capacidade para armazenar a parte fracionária do número. Se Resultado fosse Inteiro, ocorreria um truncamento ou erro. No contexto do VisuAlg e lógica da FCM, a divisão (/) produz um resultado Real.

QUESTÃO 04
Analise o trecho de algoritmo abaixo e determine o valor final da variável C. 
Var
   A, B, C : Inteiro
Inicio
   A <- 10
   B <- 20
   C <- 0
   
   Se (A > B) Entao
      C <- A + B
   Senao
      Se (A = 10) Entao
         C <- B - A
      Senao
         C <- A * B
      FimSe
   FimSe
   
   Escreva(C)
Fimalgoritmo
Resolução Detalhada:

Gabarito: C

Teste de mesa da estrutura condicional aninhada:

  1. Estado Inicial: A=10, B=20, C=0.
  2. Primeiro Teste: Se (A > B) → Se (10 > 20). Resultado: FALSO.
  3. Fluxo: O algoritmo salta para o primeiro Senao.
  4. Segundo Teste (dentro do Senao): Se (A = 10). Resultado: VERDADEIRO.
  5. Execução: Entra no bloco Entao deste segundo Se.
    • Comando: C <- B - A
    • Cálculo: 20 - 10
    • Novo valor de C: 10.
  6. Finalização: O bloco Senao do segundo Se é ignorado.
  7. Saída: Escreva(10).
QUESTÃO 05
Sobre as estruturas de repetição (laços) utilizadas na construção de algoritmos, analise as afirmativas a seguir:

I. A estrutura ENQUANTO...FACA realiza o teste lógico no início do laço. Se a condição for falsa na primeira verificação, o bloco de comandos não será executado nenhuma vez.
II. A estrutura REPITA...ATE garante que o bloco de comandos seja executado pelo menos uma vez, pois o teste lógico ocorre após a execução dos comandos.
III. A estrutura PARA...FACA é recomendada para situações onde o número de repetições é desconhecido e depende de uma condição complexa externa ao laço.

Está correto o que se afirma em:
Resolução Detalhada:

Gabarito: C

Avaliação teórica das estruturas de repetição:

  • Item I (Correto): O Enquanto é uma estrutura "Pré-Testada". Se a condição falhar na entrada, o bloco nunca roda.
  • Item II (Correto): O Repita é "Pós-Testada". Ele executa primeiro e pergunta depois, garantindo ao menos uma iteração.
  • Item III (Incorreto): O Para (For) é uma estrutura de Repetição Contada, ideal quando se sabe o número de repetições. Para números desconhecidos, o Enquanto seria a escolha correta.

Para i de 1 ate 5 faca

Gabarito: C

((X < Y) E (Y < Z)) OU (X = Z)

O resultado lógico final da avaliação desta expressão é:
Resolução Detalhada:

Gabarito: A

Resolução de expressão lógica. Valores: X=5, Y=10, Z=15.

QUESTÃO 08
Observe o algoritmo abaixo, que implementa uma estrutura de repetição Enquanto: 
Var
   Contador, Soma : Inteiro
Inicio
   Contador <- 1
   Soma <- 0
   Enquanto (Contador <= 4) Faca
      Soma <- Soma + Contador
      Contador <- Contador + 1
   FimEnquanto
   Escreva(Soma)
Fimalgoritmo
Ao final da execução, o valor impresso pela variável Soma será:
Resolução Detalhada:

Gabarito: C

Teste de mesa de acumulador. Início: Contador = 1, Soma = 0.

  • Iteração 1: Soma = 0+1 = 1 | Cont = 2
  • Iteração 2: Soma = 1+2 = 3 | Cont = 3
  • Iteração 3: Soma = 3+3 = 6 | Cont = 4

QUESTÃO 09
(Lógica de Troca de Variáveis) Analise o trecho de código a seguir: 
Var
   A, B, Aux : Inteiro
Inicio
   A <- 50
   B <- 100
   
   Aux <- A
   A <- B
   B <- Aux
Fimalgoritmo
Após a execução dos comandos acima, os valores armazenados nas variáveis A e B serão, respectivamente:
Resolução Detalhada:

Gabarito: B

Lógica clássica de Swap (Troca):

  1. Aux <- A: Aux guarda 50.
  2. A <- B: A recebe 100. (Valor antigo 50 perdido, mas salvo em Aux).
  3. B <- Aux: B recebe 50 (que estava em Aux).

Resultado: A vale 100, B vale 50.
